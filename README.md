# MouseMacro

一个基于 Python 的鼠标宏工具，支持多种操作类型、OCR 文字验证、图像匹配、动作组联动等功能。

## 特性

- **多种动作类型**：单击、长按、滚动、中键、键盘按键、拖拽、等待
- **文字验证（OCR）**：基于 RapidOCR 的文字识别，支持匹配成功/失败时的不同处理
- **图片验证**：基于 OpenCV 模板匹配，支持截图和本地上传模板
- **动作组**：将多个动作组合在一起，通过图片匹配触发执行
- **联动跳转**：动作失败时可跳转到其他步骤
- **相对窗口坐标**：支持窗口移动和分辨率缩放
- **深色主题 UI**：自定义标题栏、自定义滚动条
- **配置导入/导出**：JSON 格式持久化
- **全局热键**：一键启动/停止宏执行

## 安装

```bash
pip install -r requirements.txt
```

OCR 功能需要 `rapidocr-onnxruntime`，首次安装较大（约 1GB），可选安装。

## 运行

```bash
python main.py
```

## 打包

```bash
python -m PyInstaller --clean --noconfirm MouseMacro.spec
```

打包后的可执行文件在 `dist/MouseMacro/` 目录下。

## 安全说明

- 所有操作为纯视觉模拟：鼠标/键盘事件由 pynput 发送，OCR/图像识别通过截屏实现
- 不读取或修改任何进程内存
- 窗口检测仅使用 Win32 公开 API

## 依赖

- pynput — 鼠标/键盘模拟
- mss — 屏幕截图
- Pillow — 图像处理
- opencv-python — 模板匹配（rapidocr 的间接依赖）
- rapidocr-onnxruntime — OCR 文字识别（可选）

# 使用教程

## 一、界面概览

启动后你会看到一个深色主题的悬浮窗口，从上到下分为：

界面分为以下几个区域：

| 区域 | 说明 |
|------|------|
| **标题栏** | 显示 `⬡ 鼠标宏`，可拖拽移动窗口。右侧 `─` 最小化、`✕` 关闭 |
| **工具栏** | `▶ 运行` / `■ 停止` / `⚙ 设置`，右侧 `●` 运行状态指示灯 |
| **快捷设置栏** | 显示热键、重复次数、步骤间隔(ms)，可快速修改 |
| **动作列表** | 所有动作按序号排列，支持拖拽排序，运行时高亮当前步骤 |
| **操作按钮** | `＋ 添加` / `↑` / `↓` / `✕ 删除` / `↥ 导入` / `↧ 导出` |

| 1 | 单击 | (500,300) 50ms | V | | | ✓ |
| 2 | 键盘 | Ctrl+C | | →3 | | ✓ |

- **标题栏**：按住空白区域可拖拽移动窗口；点击 `─` 最小化，`✕` 关闭
- **工具栏**：运行/停止宏、打开全局设置
- **快捷设置栏**：快速修改热键、重复次数、步骤间隔
- **动作列表**：所有动作步骤按顺序排列，运行时高亮当前步骤
- **状态栏**：显示当前状态、操作提示

## 二、创建你的第一个宏

### 2.1 添加动作

点击 **＋ 添加** 按钮，弹出动作编辑器。从下拉菜单选择动作类型：

| 动作类型 | 用途 | 配置项 |
|---------|------|--------|
| **单击** | 移动鼠标到指定位置并点击 | 坐标 + 按住时长(ms) |
| **长按** | 移动鼠标到指定位置并长按 | 坐标 + 按住时长(ms) |
| **滚动** | 在指定位置滚动滚轮 | 坐标 + 方向 + 格数 |
| **中键** | 在指定位置点击鼠标中键 | 坐标 + 按住时长(ms) |
| **键盘** | 按下一个或多个按键 | 按键组合 + 按住时长(ms) |
| **拖动** | 从起点拖动鼠标到终点 | 起点/终点 + 时长 + 步数 |
| **等待** | 暂停执行指定时长 | 等待时长(ms) |
| **动作组** | 将多个动作打包一起执行 | 见下文 |

### 2.2 拾取坐标

在坐标区域点击 **拾取** 按钮：
1. 程序窗口会自动隐藏
2. 屏幕显示半透明覆盖层 + 绿色十字准星
3. 移动鼠标到目标位置，**左键点击** 完成拾取
4. 按 **ESC** 取消

坐标会自动回填到编辑器的 X/Y 输入框中。

### 2.3 通用设置

每个动作都有以下通用配置：
- **标签**：给动作起个名字，方便在列表中识别（会显示在列表的"描述"列）
- **执行后延迟(ms)**：该动作执行完毕后等待的时间
- **启用此步骤**：取消勾选可以暂时禁用该动作

### 2.4 运行宏

1. 添好动作后，设置热键（默认 `F6`）
2. 按热键或点击 **▶ 运行** 开始执行
3. 运行中再次按热键或点击 **■ 停止** 终止（建议用热键启动或停止而不是按键）

## 三、文字验证（OCR）

文字验证让动作在**识别到特定文字后才执行**，非常适合需要等待 UI 出现的场景。

### 3.1 配置文字验证

在动作列表中**右键**某个动作 → 选择 **文字验证**：

1. **启用文字验证**：勾选
2. **识别区域**：点击 **框选区域**，拖拽选取屏幕上文字出现的范围
3. **目标文字**：输入需要识别的文字
4. **匹配模式**：
   - **精确匹配**：识别结果必须完全等于目标文字
   - **包含**：识别结果中包含目标文字即算匹配（推荐）
   - **正则**：使用正则表达式匹配
5. **执行条件**：
   - **找到文字时执行**：文字出现才执行动作（常用）
   - **未找到文字时执行**：文字消失才执行动作
6. **条件不满足时**：
   - **跳过此步骤**：跳过当前动作，执行下一个
   - **退出脚本**：停止整个宏
   - **等待条件满足**：每隔 0.2 秒重新检测，直到文字出现或超时
7. **位置微调**：将点击位置微调到识别文字的中心（用于应对 UI 元素位置微变）
8. 点击 **测试识别** 可以立即截图该区域并显示识别结果

### 3.2 验证标记

列表中带 `V` 标记的动作表示已配置文字验证，金色文字显示。

## 四、图片验证

图片验证通过 **模板匹配** 检测屏幕上的图像，适用于那些无法用文字识别的 UI 元素（图标、按钮等）。

### 4.1 配置图片验证

在动作编辑器中，展开 **▸ 图片验证** 面板：

1. **框选区域**：拖拽选择屏幕上模板图片所在区域
2. **截图**：捕获当前屏幕区域作为模板
3. **上传图片**：也可以从本地选择一张 PNG/JPG 图片作为模板（适用于背景会变化的元素）
4. **阈值 (0.1-1.0)**：匹配置信度，越高越严格。建议从 `0.8` 开始调试
5. **失败时**：同文字验证，可选跳过/退出/等待

配置后会在编辑器中显示模板缩略图。

### 4.2 文字验证 vs 图片验证

- 同一个动作可以**同时配置**文字验证和图片验证
- 执行顺序：**先检查文字验证 → 再检查图片验证**，两者都通过才执行动作
- 列表中带 `I` 标记表示已配置图片验证

## 五、动作组

动作组用于将多个动作打包在一起，通过图片匹配触发。

### 5.1 创建动作组

1. 点击 **＋ 添加** → 选择 **动作组**
2. 展开 **▸ 图片验证** 面板，配置模板图片和阈值
3. 保存后，动作列表中会出现 📁 青色标记的动作组

### 5.2 管理子动作

在主界面中管理动作组的子动作：
- **拖入**：将列表中的普通动作拖到动作组标题行或子动作区域
- **移出**：右键子动作 → 选择 **从组中移出**
- **编辑**：双击子动作或右键 → 编辑
- **删除**：选中子动作 → 点击 ✕ 删除

### 5.3 执行逻辑

1. 执行到动作组时，先检查图片是否匹配
2. 匹配成功 → 按顺序执行组内所有子动作
3. 匹配失败 → 按"失败时"设置处理（跳过/退出/等待）
4. 子动作同样支持文字验证、图片验证、延迟等配置

### 5.4 限制

非空的动作组不能拖入其他动作组。如需嵌套，请先创建空的动作组。

## 六、联动跳转

联动让动作在**验证失败时自动跳转到另一个动作**，常用于"失败重试"场景。

### 6.1 设置联动

1. 右键某个动作 → **设置联动**
2. 在列表中选择要跳转的目标动作
3. 点击确定

配置后，当该动作的文字/图片验证失败时（且失败行为不是"退出脚本"），会**立即跳转执行**目标动作，然后再继续执行。

### 6.2 列表标记

- `→3` 表示该动作验证失败时会跳转到第 3 步
- `◆` 表示该动作是其他动作的跳转目标
- 跳转目标动作**不会在正常循环中执行**，只有被跳转时才会运行

### 6.3 典型用法：失败重试

```
1. 键盘 F     V           ← 按 F 交互
2. 等待 2000ms
3. 📁 流程 [3个子动作]      ← 检测到目标 → 执行流程
    1. 键盘 1
    2. 键盘 2
    3. 等待 1000ms
4. 📁 结算 [2个子动作]  V I  →6  ← OCR+图片双重验证
    1. 单击 (领取按钮)
    2. 等待 500ms
5. 单击 (返回按钮)                  ← 正常通关 → 返回
6. 📁 重试流程 [1个子动作]  ◆        ← 跳转目标（◆标记），正常不会执行
    1. 单击 (重新挑战按钮)
```

在第 4 步"战斗结算"未通过验证时（没打赢），自动跳到第 6 步"重试流程"点击重新挑战。

**注意**：联动跳转是在文字/图片验证失败且"失败时"设置为"跳过此步骤"时才触发。如果设置为"退出脚本"则不会跳转。

## 七、拖拽排序

- **拖动**：在列表中按住动作拖拽到目标位置
- **↑ ↓ 按钮**：选中动作后点击上下箭头移动
- **动作组整组移动**：拖动动作组标题行 → 整组（含所有子动作）一起移动
- **拖入动作组**：拖动普通动作到 📁 动作组标题或子动作区域 → 加入该组
- **从动作组移出**：拖动子动作到组外空白区域 → 提升为顶级动作

## 八、键盘录制

在键盘动作编辑器中，点击 **录制按键** 按钮：

1. 弹出录制窗口
2. 按下你要录制的按键组合（如 `Ctrl+Shift+A`）
3. 松开最后一个按键完成录制
4. 按键组合自动填入输入框

## 九、全局设置

点击 ⚙ 图标打开全局设置：

| 设置项 | 说明 |
|--------|------|
| 启动/停止热键 | 全局快捷键，默认 `F6` |
| 重复次数 | 0 = 无限循环，其他数字表示执行轮数 |
| 步骤间隔(ms) | 每个动作之间的基础等待时间 |
| 随机延迟 | 在基础间隔上额外随机增加 N~M 毫秒的延迟 |
| 出错时停止 | 勾选后，任何动作执行失败都会终止宏 |
| 鼠标移动速度(ms) | 平滑移动鼠标的时长，越大越慢（适合某些检测鼠标速度的游戏） |

### 高级坐标功能

展开 **▸ 高级坐标功能（相对窗口）**：

- **目标窗口标题**：通过拾取窗口功能自动获取
- **锁定**：勾选后每 3 秒自动检测窗口分辨率变化并更新参考值
- **参考分辨率**：拾取窗口时自动填充
- **批量切换**：一键将所有动作在绝对坐标和相对窗口模式间切换

## 十、导入导出

- **↧ 导出**：将当前所有动作和设置保存为 `.json` 文件
- **↥ 导入**：加载之前保存的 `.json` 配置
- JSON 文件可跨设备使用，配合相对窗口坐标可实现分辨率自适应

## 十一、常见问题

**Q: 为什么宏执行时游戏画面会偏移？**
A: 鼠标从当前位置瞬移到目标坐标时，某些游戏会将其识别为视角拖动。解决方案：
1. 在全局设置中增大"鼠标移动速度"
2. 避免在拖拽动作后立即进行大幅度的坐标移动

**Q: OCR 显示"不可用"？**
A: 需要安装 `rapidocr-onnxruntime`：`pip install rapidocr-onnxruntime`。首次加载 OCR 模型比较慢，启动时会后台预热。

**Q: 如何让宏适应不同分辨率？**
A: 在全局设置中启用高级坐标功能 → 拾取目标窗口 → 使用相对窗口坐标模式。详情见第九节。

**Q: 图片验证总是匹配不上？**
A: 尝试降低阈值（如 0.6），确认模板区域包含了完整的 UI 元素，或者使用本地图片上传一个干净的模板图。
